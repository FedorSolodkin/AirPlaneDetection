# Обнаружение самолётов — YOLO на HRPlanes

Беcякорный детектор самолётов в стиле YOLO для спутниковых снимков высокого разрешения
(датасет HRPlanes). Реализован на PyTorch с нуля — без `ultralytics`, без `mmdet`.
Backbone ResNet-18 (torchvision, предобученный на ImageNet) передаёт карту признаков
в небольшой neck (свёртка 3x3) и head (свёртка 1x1), который предсказывает на каждой
ячейке сетки с stride 16: objectness + (cx, cy, w, h) + логиты классов.

## Структура проекта

    configs/
      config.yaml         -- гиперпараметры данных / модели / функции потерь / обучения / оценки
    src/
      dataset.py          -- Dataset в формате YOLO + collate_fn
      transform.py        -- letterbox, пересчёт меток, аугментации
      model.py            -- ResNet backbone + neck + голова детектора
      loss.py             -- objectness (BCE) + бокс (CIoU) + класс (BCE)
      metrics.py          -- decode_predictions (conf + NMS) + mAP@0.5
      utils.py            -- конфиг / устройство / чекпоинты / математика боксов
    scripts/
      download.py         -- загрузка HRPlanes с Zenodo, распаковка в data/hrplanes
      train.py            -- запуск обучения
      val.py              -- оценка чекпоинта на валидационной выборке
      test.py             -- оценка чекпоинта на тестовой выборке

## Датасет

HRPlanes — 3092 изображения VHR (4800x2703) с крупных аэропортов, ~18k разметок
самолётов, формат меток YOLO, разбивка 70/20/10. Запись Zenodo `14546832`
(см. https://zenodo.org/records/14546832).

Скачать и распаковать одной командой:

    python scripts/download.py

Создаёт структуру папок, ожидаемую `configs/config.yaml`:

    data/hrplanes/
      train/images/*.jpg    train/labels/*.txt
      val/images/*.jpg      val/labels/*.txt
      test/images/*.jpg     test/labels/*.txt

Для HRPlanesv2 используйте `HRPLANES_RECORD=7331974 python scripts/download.py`.

## Обучение

    python scripts/train.py --config configs/config.yaml

Чекпоинты сохраняются в `assets/models/`:
  - `last.pt` — последняя эпоха
  - `best.pt` — лучший mAP@0.5 на валидационной выборке

## Оценка

    python scripts/val.py  --ckpt assets/models/best.pt
    python scripts/test.py --ckpt assets/models/best.pt

## Математика модели (ключевые детали)

**Backbone.** Torchvision ResNet-18, урезанный до `layer3` — выход имеет stride 16
и 256 каналов. По умолчанию загружаются веса, предобученные на ImageNet.

**Neck + head.** Свёртка 3x3 + BatchNorm + SiLU обрабатывает карту признаков, затем
свёртка 1x1 выдаёт `5 + num_classes` каналов на ячейку:
  - 1 канал логита objectness
  - 4 канала смещений бокса, пропущенных через sigmoid, чтобы (cx, cy, w, h)
    лежали в [0, 1] относительно входного изображения
  - C каналов логитов классов

Смещение (bias) objectness инициализируется значением -4.0, чтобы сеть с самого
начала была почти «молчаливой» — это стабилизирует раннее обучение.

**Назначение целей** (`src/loss.py::build_targets`). Каждый GT-бокс назначается
единственной ячейке, содержащей его центр в координатах сетки. Эта ячейка
обучается по трём потерям; все остальные ячейки являются негативными примерами.

**Функция потерь:**
  - obj  : взвешенный BCE на логит objectness. Позитивные ячейки получают вес `w_obj`,
           негативные — `w_noobj` (по умолчанию 0.5), чтобы сетка 40x40 не «заглушала»
           немногочисленные позитивные примеры.
  - bbox : 1 − CIoU между предсказанными и целевыми боксами на совпавших ячейках.
  - cls  : BCE (в стиле multi-label) на логиты классов совпавших ячеек.

Все три суммируются с весами из `configs/config.yaml`.

**Декодирование** (`src/metrics.py::decode_predictions`). Для каждого изображения
уверенность каждой ячейки вычисляется как `sigmoid(obj) * sigmoid(cls)`; ячейки
ниже `conf_thresh` отбрасываются, сохраняются top-K выживших, затем применяется
NMS по каждому классу.

**mAP.** `compute_map50` сортирует детекции по уверенности для каждого класса,
строит PR-кривую и усредняет точность в 101 равноудалённой точке отзыва
(интерполяция mAP@0.5 в стиле COCO). Для одного класса результат — это один AP.

**Оптимизатор.** AdamW с двумя группами параметров: backbone обучается со скоростью
`lr_backbone` (в 10 раз меньше), остальное — со скоростью `lr0`. Расписание LR
по косинусному затуханию, обрезка градиентов по норме `clip_max_norm`, смешанная
точность через `torch.cuda.amp`.

## Примечания

- Никакого `ultralytics`. Каждая часть пайплайна — датасет, аугментации,
  модель, назначение целей, функция потерь, декодер, mAP — находится в `src/`.
- Для GPU с небольшим объёмом памяти установите `data.batch_size: 4` или
  `data.imgsz: 512` в `configs/config.yaml`.
- `src/metrics.py` вычисляет mAP@0.5 со 101-точечной интерполяцией по каждому классу.
  Для полного COCO-style mAP@[.5:.95] используйте torchmetrics или pycocotools —
  для однокласcового HRPlanes в этом нет необходимости.
